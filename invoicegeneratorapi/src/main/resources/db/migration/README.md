# Database Migration Guide

## Migration V2: Invoice Auto Status Sync Fields

### Overview
This migration adds fields required for the Invoice Auto Status Sync feature, which implements automated invoice lifecycle management.

### Migration File
`V2__add_invoice_auto_status_sync_fields.js`

### Fields Added

#### Status Tracking Fields (Requirements 8.1-8.6)
- `status` (ENUM): Invoice status with values DRAFT, SENT, VIEWED, PAID, OVERDUE, CANCELLED
- `isAutoGeneratedStatus` (BOOLEAN): Flag indicating if status was auto-generated (default: TRUE)
- `lastStatusUpdatedAt` (TIMESTAMP): Last time the status was updated
- `emailSentAt` (TIMESTAMP): When the invoice email was sent
- `firstViewedAt` (TIMESTAMP): When the invoice was first viewed by customer
- `paidAt` (TIMESTAMP): When the invoice was paid

#### Razorpay Integration Fields (Requirements 8.7-8.9)
- `razorpayPaymentLinkId` (VARCHAR): Razorpay payment link identifier
- `razorpayOrderId` (VARCHAR): Razorpay order identifier
- `razorpayPaymentId` (VARCHAR): Razorpay payment identifier

#### Payment Status Field (Requirement 8.10)
- `paymentStatus` (ENUM): Payment status with values PENDING, SUCCESS, FAILED

### Migration Actions

1. **Initialize Status Field**: Sets status to DRAFT for invoices without status
2. **Initialize Auto-Generated Flag**: Sets isAutoGeneratedStatus to TRUE for all invoices
3. **Initialize Timestamp Fields**: Sets all timestamp fields to null for invoices that don't have them
4. **Migrate Nested Payment Data**: Moves Razorpay fields from nested PaymentDetails to top level
5. **Initialize Razorpay Fields**: Sets Razorpay fields to null for invoices that don't have them
6. **Initialize Payment Status**: Sets paymentStatus to PENDING for invoices without it
7. **Update Payment Status for Paid Invoices**: Sets paymentStatus to SUCCESS for invoices with PAID status
8. **Migrate Legacy Payment Status Values**: Converts PAID to SUCCESS and REFUNDED to FAILED
9. **Set Last Status Updated Timestamp**: Uses lastUpdatedAt as fallback for lastStatusUpdatedAt

### Running the Migration

#### Using MongoDB Shell
```bash
mongosh mongodb://localhost:27017/Invoizo < V2__add_invoice_auto_status_sync_fields.js
```

#### Using MongoDB Compass
1. Connect to your MongoDB instance
2. Select the `Invoizo` database
3. Open the MongoDB shell in Compass
4. Copy and paste the migration script
5. Execute the script

### Verification

The migration script includes built-in verification that checks:
- All invoices have the required fields
- All status values are valid (DRAFT, SENT, VIEWED, PAID, OVERDUE, CANCELLED)
- All payment status values are valid (PENDING, SUCCESS, FAILED)
- Displays a sample invoice structure after migration

### Rollback

If you need to rollback this migration:

```javascript
// Remove new fields
db.invoices.updateMany(
    {},
    {
        $unset: {
            isAutoGeneratedStatus: "",
            lastStatusUpdatedAt: "",
            emailSentAt: "",
            firstViewedAt: "",
            paidAt: "",
            razorpayPaymentLinkId: "",
            razorpayOrderId: "",
            razorpayPaymentId: "",
            paymentStatus: ""
        }
    }
);

// Note: status field should be kept as it may be used by existing code
```

### Notes

- This migration is idempotent and can be run multiple times safely
- Existing invoices will have their status set to DRAFT by default
- Payment status will be set to PENDING by default
- The migration preserves existing data in nested PaymentDetails while adding top-level fields
- All timestamp fields are initialized to null and will be populated by the application as events occur
